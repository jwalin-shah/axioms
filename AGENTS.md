# Axioms — Agent Instructions

**Project:** Axiom corpus — 2183 engineering axioms from production systems, textbooks, and standards.
**Role:** Reference library for bridge + orbit verification. Not a codebase — a knowledge base.
**Default branch:** `main`

---

## What This Project Is

Axioms is the source of truth for engineering invariants. It feeds Neo4j
(knowledge engine) via cocoindex pipelines. Bridge and orbit consume it through
`bridge/knowledge/` Go adapter at context assembly time, and through the audit
workflow for pre-spawn verification.

Three layers:
1. **Literature axioms** (`axioms.json`) — claims from books, standards, papers. 2177 VERIFIED, 1 UNCERTAIN, 5 DISPUTED.
2. **Source cache** (`source-cache/`) — the original text each axiom was extracted from. Required for verification.
3. **Guidance** (`guidance/`) — non-∀-provable rules that need different enforcement (linters, OpenAPI schemas, code review).

## Critical Rules

### Source trust levels (from 825-axiom adversarial verification run)

| Source Type | Verify Rate | Trust |
|---|---|---|
| `textbook-formal` (CLRS, Sedgewick, TAOCP, Okasaki, TAPL) | 84% | HIGH |
| `oracle-extract` (cached MD files) | 60% | MEDIUM |
| `guideline` | 75% | MEDIUM |
| `textbook-research` (Anderson, Schneier, Katz) | 25% | LOW |
| `standard` (NIST, ISO, FIPS) | 14% | VERY LOW |
| `research-primary` (no citation, Tier C) | 0% | NO TRUST |
| `house-rule` (our own rules) | 0% | NO TRUST — must review |

### P0 rules (hard blocks)
- Never verify an uncited axiom. Cache the source first.
- Never extract invariants from guidance literature. Standards/best practices can't be rendered as ∀ equations.
- Extracted equations must not be truncated. If source is on disk, re-extract; don't re-verify a truncated equation.

### P1 rules (must include evidence)
- Hand-written rules need adversarial review before promotion.
- Source type is the trust signal. Check it before using an axiom to make a claim.
- An axiom is not verified until there's a TestAX* gate that proves it holds for the code.

## Audit Workflow

The audit workflow (`bridge-orbit-audit`) runs against bridge + orbit using axioms as the standard. It:

1. **Scouts** the axiom corpus for relevant categories (Saltzer-Schroeder, software-correctness, software-testing, architecture, sandbox, safety, provenance, testing, testability, systems)
2. **Audits bridge** across three known findings (#7 delivery control, #8 sandbox fail-open, #9 verify-machine blind spot) plus general correctness
3. **Audits orbit** across dispatch, session isolation, and orchestration
4. **Adversarially verifies** each finding — a separate agent tries to refute it
5. **Synthesizes** a report with confirmed findings and recommendations

The audit is launched from this repo because it owns the axiom corpus. It reads bridge and orbit code from their respective repos but does not modify them.

**To run the audit:**
```bash
# Neo4j must be running with current axiom data
brew services list | grep neo4j  # verify running

# The audit is launched via Workflow from the axioms directory
# It reads /tmp/audit_final.json (generated by the pre-audit axiom filter)
```

**Filtering methodology:** Not all 2183 axioms apply to bridge/orbit. The pre-audit filter selects axioms by:
1. **Category match:** architecture, sandbox, software-correctness, software-testing, testing, testability, provenance, systems, safety, Saltzer-Schroeder
2. **Concept search:** fencing tokens, isolation, sandbox, privilege, capability, audit log, ledger, provenance, non-repudiable, dispatch, orchestration, seccomp, hermetic, scope
3. **Exclusion:** Protocol-level axioms (TCP state machines, HTTP chunked encoding, TLS handshakes) are excluded — they're network specs, not software design principles

## Neo4j

The knowledge engine runs on `neo4j://localhost:7687` (neo4j/axiom-knowledge).
Query it directly:
```bash
curl -s -u neo4j:axiom-knowledge "http://localhost:7474/db/neo4j/tx/commit" \
  -H "Content-Type: application/json" \
  -d '{"statements":[{"statement":"MATCH (a:Axiom) RETURN count(a)"}]}'
```

Bridge queries it via `bridge/knowledge/` Go package. The audit queries it via curl.
